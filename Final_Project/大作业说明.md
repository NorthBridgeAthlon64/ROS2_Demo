## 一、机器人模拟器代码详细解释

### 整体结构

```
robot_simulator.py 做了三件事：
1. 模拟机器人运动（收到 /cmd_vel → 更新位置）
2. 模拟传感器（计算激光雷达每束射线打到哪）
3. 发布数据（/odom 和 /scan）
```

### 逐段解释

```python
class RobotSimulator(Node):
    def __init__(self):
        super().__init__('robot_simulator')
        
        # ====== 机器人当前状态（三个数决定机器人在哪） ======
        self.x = 1.0          # 世界坐标 X（米）
        self.y = 1.0          # 世界坐标 Y（米）
        self.theta = 0.0      # 朝向（弧度），0=朝右
```

**这三个数就是机器人的“数字孪生”**——任何时候，知道 `(x, y, θ)` 就知道机器人在哪、面朝哪。

---

### 运动模型（最核心的物理）

```python
def update(self):
    # 差速模型：给定线速度和角速度，更新位置
    self.x += self.vx * math.cos(self.theta) * self.dt
    self.y += self.vx * math.sin(self.theta) * self.dt
    self.theta += self.vw * self.dt
```

**图解**：

```
时刻 t 的位置                    时刻 t+dt 的位置
                                    
    y ↑                              y ↑
      │                                │
      │   🤖 (x, y)                     │        🤖 (x', y')
      │   → 朝向 θ                      │        → 朝向 θ'
      │                                │       ╱
      │   vx = 0.3 m/s                 │      ╱ 走了 vx×dt 这么远
      │                                │     ╱
      └────────────→ x                └────────────→ x

新位置:
  x' = x + vx × cos(θ) × dt    ← 速度分解到 X 轴
  y' = y + vx × sin(θ) × dt    ← 速度分解到 Y 轴
  θ' = θ + vw × dt             ← 角速度积分
```

**生活比喻**：你骑自行车，蹬得快（vx 大）就走得远，车把往左转（vw 不为0）方向就变。每 0.1 秒算一次新位置。

### 激光雷达模拟（光线投射）

```python
def ray_cast(self, angle):
    distance = 0.0
    while distance < 5.0:                    # 最远看5米
        ray_x = self.x + distance * cos(angle)  # 射线端点的X
        ray_y = self.y + distance * sin(angle)  # 射线端点的Y
        row, col = world_to_grid(ray_x, ray_y)  # 转成地图格子
        if get_cell(row, col) == 1:             # 碰到墙了
            return distance                      # 返回这个距离
        distance += 0.1                          # 没碰到，再往前探0.1米
    return 5.0  # 到头了，没障碍
```

**图解**：

```
机器人发射一束激光，方向 angle，逐步探测：

         🤖──────────→──────────→────🧱
          │          │          │
    distance=0    distance=1  distance=2.3
          │          │          │
    get_cell=空地  get_cell=空地  get_cell=墙！
                                  │
                                 返回 2.3（米）
```

360 束激光，每束都这样算，拼起来就是 `/scan` 话题里的 360 个距离值。

## 二、为什么没用 URDF/Xacro 定义机器人

**URDF 是给 Gazebo 和 rviz2 看的。我们不用它们，所以不需要 URDF。**

| 需要什么信息 | URDF 方式 | 纯代码方式 |
|-------------|---------|----------|
| 机器人长什么样 | XML 描述长方体、圆柱体 | 不需要（不画 3D） |
| 轮子在哪 | joint 定义位置 | `wheel_base = 0.3` 一个数 |
| 激光雷达在哪 | link + joint | 激光从 `(x, y)` 出发，方向 `θ` |
| 有多重 | inertial 标签 | 不需要（没做动力学） |

**纯代码把机器人抽象成了三个关键数据**：

```python
self.x, self.y, self.theta    # 位置和朝向（代替了 URDF 的 base_link）
self.wheel_base = 0.3         # 轮距（代替了 URDF 的 wheel joints）
# 激光雷达的位置 = 机器人位置 + 朝向前方（隐式定义，不需要 joint）
```

## 三、BFS 路径规划算法详解

### 要解决的问题

```
地图上：S = 起点(机器人)，G = 终点(货架)，# = 墙

   0  1  2  3  4
0  S  ·  ·  ·  ·
1  ·  #  ·  ·  ·
2  ·  #  ·  G  ·
3  ·  ·  ·  ·  ·

找一条从 S 到 G、不穿墙的最短路径
```

### BFS 核心思想

**层层扩散，像水波一样**

```
第0步（起点）：             第1步（走1格能到的）：
┌──────────┐              ┌──────────┐
│ S  ·  · │              │ S  1  · │
│ ·  #  · │              │ 1  #  · │
│ ·  ·  G │              │ ·  ·  G │
└──────────┘              └──────────┘

第2步（走2格能到的）：      第3步（到终点了！）：
┌──────────┐              ┌──────────┐
│ S  1  2 │              │ S  1  2 │
│ 1  #  2 │              │ 1  #  2 │
│ 2  ·  G │              │ 2  3  G │  ← 找到！
└──────────┘              └──────────┘

回溯路径：G ← (2,3) ← (1,3) ← (0,2) ← (0,1) ← (1,0) ← S
         终点    第3步    第2步    第1步    第1步    起点
```

### 代码和图的对应

```python
def bfs(self, start_row, start_col, goal_row, goal_col):
    # 队列：先进先出，保证"先发现的先处理"
    queue = deque()
    queue.append((start_row, start_col, []))  # (行, 列, 走到这的路径)
    visited = set()
    visited.add((start_row, start_col))
    
    while queue:
        row, col, path = queue.popleft()  # 取出队首
        
        if row == goal_row and col == goal_col:
            return path + [(row, col)]   # 到了！
        
        for dr, dc in [(-1,0),(1,0),(0,-1),(0,1)]:  # 上下左右
            new_row = row + dr
            new_col = col + dc
            
            if (new_row, new_col) not in visited:
                if get_cell(new_row, new_col) != 1:  # 不是墙
                    visited.add((new_row, new_col))
                    queue.append((new_row, new_col, path + [(row, col)]))
    
    return None  # 没路
```

### 队列变化过程

```
初始：queue = [(S, [])]

第1轮：取出S，发现上下左右4个邻居
      queue = [(S上, [S]), (S下, [S]), (S左, [S]), (S右, [S])]

第2轮：取出S上，发现它的邻居
      queue = [(S下, [S]), (S左, [S]), (S右, [S]), (S上上, [S,S上]), ...]

第3轮：取出S下...

一直往外扩，直到碰到 G
```

**关键**：因为是一层层往外扩，第一个到达 G 的路径一定是**最短路径**。

---

### BFS vs A* 对比

| | BFS | A* |
|------|-----|-----|
| 探索方式 | 所有方向均匀扩散 | 优先往目标方向探索 |
| 知道目标在哪吗 | ❌ 不知道，盲目扩散 | ✅ 知道，用启发函数引导 |
| 速度 | 慢（探索很多无关格子） | 快（少探索很多格子） |
| 保证最短路径 | ✅ 保证 | ✅ 保证（H 不高估时） |
| 适合场景 | 小地图、没方向感 | 大地图、目标明确 |

**你的大作业用 BFS 就够了，地图 20×20，BFS 瞬间算完。** 学生如果想优化，可以自己改成 A*。

大作业里没有“画”3D 模型，而是用**几个关键变量**定义了机器人的物理特性。

## 运动模型（怎么“动”起来）

```python
# 差速驱动模型
self.x += self.vx * cos(self.theta) * self.dt   # 把速度分解到 X 轴
self.y += self.vx * sin(self.theta) * self.dt   # 把速度分解到 Y 轴
self.theta += self.vw * self.dt                 # 角速度积分
```

```
前进时的速度分解：

                    前进方向（vx）
                   ╱
                  ╱
                 ╱ θ
                🤖────────→
                │         ↑
                │         │  vx × sin(θ)  ← Y 轴分量
                │         │
                │         │
                │         └── vx × cos(θ)  ← X 轴分量

转弯时：
    左轮转得慢 + 右轮转得快 = 向左转
    左轮转得快 + 右轮转得慢 = 向右转
    
    简化成：self.theta += vw × dt
```

## 碰撞检测（不能穿墙）

```python
row, col = world_to_grid(self.x, self.y)  # 把世界坐标转成地图格子
if get_cell(row, col) == 1:               # 这个格子是墙？
    # 弹回去！
    self.x -= vx * cos(θ) * dt
    self.y -= vx * sin(θ) * dt
```

```
碰撞检测示意图：

    移动前                    移动后（穿墙了！）          弹回来

    🤖 · · · █              · · · 🤖 █              🤖 · · · █
            墙                       墙                        墙

机器人说："新位置在墙里？不对，退回去。"
```

## 和真实物理模型对比

| 属性 | 真机器人 | 大作业模型 |
|------|---------|-----------|
| 位置 | 里程计/编码器 | `x, y` |
| 朝向 | 陀螺仪 | `theta` |
| 转弯特性 | 两个电机差速驱动 | `wheel_base` + 差速公式 |
| 碰撞 | 物理碰撞 | 检查地图格子 |
| 速度 | 电机转速反馈 | `vx, vw` |

**真机器人的状态是传感器测出来的，大作业是算出来的。但数值格式完全相同。**

## 一句话总结

> 大作业没有画 3D 模型，而是用一个点 `(x, y)` + 一个角度 `θ` + 差速驱动公式，描述机器人在 2D 世界的全部物理行为。Gazebo 把这个点画成 3D 形状，rviz2 把这个点画成箭头，我们把这个点画成 🤖。**核心数据一模一样，只是呈现方式不同。**
